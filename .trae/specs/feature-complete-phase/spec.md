# 蓝色星原战斗排轴编辑器 - 功能完善期 - 产品需求文档

## Overview
- **Summary**: 本项目是《蓝色星原：旅谣》游戏的战斗排轴编辑器，第三阶段（功能完善期）将完成核心功能升级，实现「排轴设计→合法性校验→伤害计算→数据可视化→分享传播」的完整闭环，1:1对齐参考原型Endaxis的核心专业能力。
- **Purpose**: 帮助硬核玩家实现精确的战斗排轴设计，通过完整的伤害计算、合法性校验和数据可视化，提供专业的排轴优化工具。
- **Target Users**: 《蓝色星原：旅谣》硬核玩家，需要优化战斗策略的核心玩家群体。

## Goals
- 完成完整伤害计算引擎升级，适配游戏全乘区伤害公式和核心机制
- 实现循环合法性自动校验系统，检测CD冲突、资源缺口、技能依赖等问题
- 升级排轴可视化能力，实现技能连携SVG贝塞尔曲线、buff生效区间和资源变化曲线
- 完成完整数据统计与可视化分析，基于ECharts实现全维度数据图表展示
- 开发内置游戏数据编辑器，支持新增/修改/删除角色、技能、奇波、敌人数据
- 实现项目导入导出与社区分享体系，支持专属格式、图片、Markdown报告导出
- 补全编辑器核心能力，包括撤销/重做、关键帧标记、循环播放预览等
- 优化全局设置与体验，完善主题切换、新手引导和使用教程
- 补全游戏数据，完成8个以上核心角色和4个以上高难Boss的完整数据录入

## Non-Goals (Out of Scope)
- 在线实时协作编辑功能
- 移动端完整适配（保持基础响应式布局）
- 多语言完全本地化（仅保留中英文基础支持）
- 复杂的AI排轴推荐功能
- 游戏内实时同步功能

## Background & Context
- 项目基于第二阶段（MVP核心开发期）已完成的代码架构，使用Vue 3 + Vite + Pinia + Element Plus技术栈
- 参考原型为Endaxis（《明日方舟：终末地》排轴工具），需要对齐其核心专业能力
- 游戏数据基于《蓝色星原：旅谣》的战斗机制，需要确保数据准确性和机制适配性

## Functional Requirements
- **FR-1**: 完整伤害计算引擎升级 - 补全游戏全乘区伤害公式，适配印记体系、韧性破防、DOT持续伤害、技能快照、buff联动、元素共鸣等全部核心机制
- **FR-2**: 循环合法性自动校验系统 - 完成CD冲突、资源缺口、技能依赖、触发条件的全维度自动校验，生成可视化校验报告
- **FR-3**: 排轴可视化能力升级 - 完成技能连携依赖的SVG贝塞尔曲线可视化、buff/debuff生效区间可视化、星决能量/印记/专属资源的实时变化曲线绘制
- **FR-4**: 完整数据统计与可视化分析 - 完成DPS时间曲线、伤害构成分析、buff覆盖率统计、资源变化统计，基于ECharts实现全维度数据图表展示
- **FR-5**: 内置游戏数据编辑器 - 1:1复刻参考原型的可视化数据编辑器，支持新增/修改/删除角色、技能、奇波、敌人数据，一键生成并下载gamedata.json文件
- **FR-6**: 项目导入导出与社区分享体系 - 完成专属格式导出/导入、排轴图高清图片导出、Markdown报告导出、分享链接生成、预设轴库完整功能
- **FR-7**: 编辑器核心能力补全 - 完成技能块拆分/合并、撤销/重做历史记录、关键帧标记、循环播放预览、轨道自定义、快捷键体系完善
- **FR-8**: 全局设置与体验优化 - 完成编辑器个性化设置、深色/浅色主题完善、新手分步引导、使用教程文档、全局异常捕获与错误提示
- **FR-9**: 游戏数据补全 - 完成8个以上核心角色的完整数据录入、4个以上高难Boss的完整机制与数值录入，覆盖游戏主流配队体系

## Non-Functional Requirements
- **NFR-1**: 性能 - 技能块拖拽流畅，响应时间 < 100ms，大项目加载时间 < 2s
- **NFR-2**: 数据准确性 - 伤害计算与实机误差 ≤ 2%
- **NFR-3**: 代码质量 - 代码通过ESLint检查，无警告、无错误，注释完整
- **NFR-4**: 可维护性 - 代码结构清晰，逻辑模块化，无冗余依赖
- **NFR-5**: 兼容性 - 支持主流浏览器（Chrome、Firefox、Edge），响应式布局适配PC端

## Constraints
- **Technical**: 基于第二阶段的项目架构，使用Vue 3 + Vite + Pinia + Element Plus，禁止重构核心架构
- **Business**: 3周开发周期，优先保证P0核心功能
- **Dependencies**: 依赖Vue.Draggable实现拖拽功能，依赖ECharts实现数据可视化，依赖SVG实现技能连携可视化

## Assumptions
- 游戏数据基于玩家社区实测数据和官方资料，保证准确性
- 伤害计算引擎的实现基于游戏内机制的公开信息和玩家分析
- 项目主要面向PC端用户，响应式布局优先适配PC端

## Acceptance Criteria

### AC-1: 完整伤害计算引擎
- **Given**: 用户在编辑器中添加技能块
- **When**: 用户查看伤害计算结果
- **Then**: 系统显示完整的伤害计算结果，包括印记体系、韧性破防、DOT持续伤害等所有核心机制的计算
- **Verification**: `programmatic`
- **Notes**: 伤害计算与实机误差 ≤ 2%

### AC-2: 循环合法性校验
- **Given**: 用户在编辑器中编辑排轴
- **When**: 系统自动进行合法性校验
- **Then**: 系统标记CD冲突、资源缺口、技能依赖等问题，并生成校验报告
- **Verification**: `programmatic`
- **Notes**: 支持点击问题项一键跳转到时间轴对应位置

### AC-3: 技能连携可视化
- **Given**: 用户在编辑器中创建有依赖关系的技能块
- **When**: 系统渲染时间轴
- **Then**: 系统使用SVG贝塞尔曲线连接有依赖关系的技能块，颜色区分满足状态
- **Verification**: `human-judgment`
- **Notes**: 支持鼠标悬浮显示依赖详情，点击曲线选中两端技能块

### AC-4: 数据统计与可视化
- **Given**: 用户完成排轴编辑
- **When**: 用户查看统计面板
- **Then**: 系统显示DPS时间曲线、伤害构成分析、buff覆盖率统计等全维度数据图表
- **Verification**: `human-judgment`
- **Notes**: 支持图表缩放、悬浮查看详情、一键导出高清图片

### AC-5: 游戏数据编辑器
- **Given**: 用户访问数据编辑器页面
- **When**: 用户编辑角色、技能、敌人数据
- **Then**: 系统支持新增/修改/删除数据，并一键生成下载gamedata.json文件
- **Verification**: `programmatic`
- **Notes**: 支持数据合法性校验和版本管理
