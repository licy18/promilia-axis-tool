# 蓝色星原战斗排轴编辑器 - MVP核心开发期 - 产品需求文档

## Overview
- **Summary**: 本项目是《蓝色星原：旅谣》游戏的战斗排轴编辑器，第二阶段（MVP核心开发期）将完成核心功能开发，实现「新建项目→选择角色→拖拽排轴→计算伤害→保存项目」的完整流程闭环。
- **Purpose**: 帮助玩家优化战斗策略，提高游戏体验，通过可视化编辑和伤害计算，实现精确的技能释放时序规划。
- **Target Users**: 《蓝色星原：旅谣》玩家，特别是需要优化战斗策略的核心玩家。

## Goals
- 完成角色与技能数据管理模块，支持角色列表展示和技能详情查看
- 实现项目初始化与管理功能，支持新建、保存、打开项目
- 开发核心时间轴编辑器框架，支持多轨道编辑和精确到帧的时间网格
- 实现技能块拖拽与基础编辑功能，支持技能块的添加、调整和编辑
- 开发基础伤害计算引擎，实现游戏伤害机制的模拟计算
- 实现项目本地保存与读取功能，支持项目的导出和导入
- 优化全局基础体验，包括主题切换和响应式布局

## Non-Goals (Out of Scope)
- 在线社区功能和预设分享
- 高级伤害计算（如元素反应、复杂 buff 交互）
- 完整的武器和装备系统
- 多人协作编辑功能
- 移动端适配（优先 PC 端）

## Background & Context
- 项目基于第一阶段搭建的骨架，使用 Vue 3 + Vite + Pinia + Element Plus 技术栈
- 参考原型为 Endaxis（《明日方舟：终末地》排轴工具），需要对齐其核心交互逻辑
- 游戏数据基于《蓝色星原：旅谣》的战斗机制，需要确保数据准确性

## Functional Requirements
- **FR-1**: 角色与技能数据管理 - 完成3个核心角色（末音、洛卿、源千代）的完整数据录入，支持角色列表展示和详情查看
- **FR-2**: 项目初始化与管理 - 支持新建排轴项目，选择上场角色和敌人，展示项目列表
- **FR-3**: 核心时间轴编辑器 - 实现三栏布局，包含技能库、多轨道时间轴和属性面板
- **FR-4**: 技能块拖拽与编辑 - 支持技能块的拖拽、调整、编辑和批量操作
- **FR-5**: 基础伤害计算 - 实现游戏基础伤害乘区计算，支持单技能和排轴总伤害统计
- **FR-6**: 项目本地保存与读取 - 支持自动保存、手动保存、项目导出和导入
- **FR-7**: 全局体验优化 - 支持主题切换、基础快捷键和响应式布局

## Non-Functional Requirements
- **NFR-1**: 性能 - 技能块拖拽流畅，响应时间 < 100ms
- **NFR-2**: 数据准确性 - 伤害计算与实机基础误差 < 5%
- **NFR-3**: 代码质量 - 代码通过 ESLint 检查，无警告、无错误
- **NFR-4**: 可维护性 - 代码结构清晰，注释完整，遵循最佳实践

## Constraints
- **Technical**: 基于第一阶段的项目架构，使用 Vue 3 + Vite + Pinia + Element Plus
- **Business**: 3周开发周期，优先保证 P0 核心功能
- **Dependencies**: 依赖 Vue.Draggable 实现拖拽功能，依赖 ECharts 实现伤害统计图表

## Assumptions
- 游戏数据基于玩家社区实测数据，保证基础准确性
- 伤害计算只考虑基础乘区，不考虑复杂的元素反应和 buff 交互
- 项目主要面向 PC 端用户，响应式布局优先适配 PC 端

## Acceptance Criteria

### AC-1: 角色与技能数据管理
- **Given**: 用户打开游戏数据图鉴页
- **When**: 用户点击角色列表中的角色
- **Then**: 右侧显示该角色的详细信息，包括基础属性和技能数据
- **Verification**: `human-judgment`

### AC-2: 项目初始化
- **Given**: 用户在首页点击「新建排轴项目」按钮
- **When**: 用户填写项目信息并选择角色和敌人
- **Then**: 系统创建新项目并进入编辑器页面
- **Verification**: `programmatic`

### AC-3: 技能块拖拽
- **Given**: 用户在技能库中选择一个技能
- **When**: 用户将技能拖拽到时间轴的对应轨道
- **Then**: 技能块出现在时间轴上，位置正确
- **Verification**: `human-judgment`

### AC-4: 伤害计算
- **Given**: 用户在时间轴上添加技能块
- **When**: 用户查看右侧属性面板
- **Then**: 面板显示技能的伤害数据和排轴总伤害
- **Verification**: `programmatic`

### AC-5: 项目保存与读取
- **Given**: 用户在编辑器中完成排轴编辑
- **When**: 用户点击保存按钮，然后关闭并重新打开项目
- **Then**: 项目数据完整还原，包括技能块位置和配置
- **Verification**: `programmatic`

### AC-6: 主题切换
- **Given**: 用户在编辑器中点击主题切换按钮
- **When**: 用户选择不同的主题
- **Then**: 系统切换到对应的主题，界面样式更新
- **Verification**: `human-judgment`
