# 技能系统扩展 - 产品需求文档

## Overview
- **Summary**: 为promilia-axis-tool中的技能系统增加判定系统、时序系统和状态效果系统，使其能够在时间轴上显示伤害判定点、状态持续时间条和buff持续时间条。
- **Purpose**: 提供更精确的技能效果可视化和计算，参考Endaxis中的技能写法，提升排轴编辑器的专业性和使用体验。
- **Target Users**: 游戏玩家、排轴爱好者、攻略创作者。

## Goals
- 实现技能伤害判定点系统，在时间轴上显示精确的伤害触发时刻
- 实现时序系统，包括buff持续时间条和技能冷却时间条
- 实现状态效果系统，支持增加或消耗印记等状态效果
- 在时间轴上可视化显示所有这些元素
- 确保与现有技能系统的兼容性

## Non-Goals (Out of Scope)
- 重新设计整个技能编辑器界面
- 修改现有的技能数据结构（保持向后兼容）
- 实现实时战斗模拟
- 添加新的角色或敌人数据

## Background & Context
- 参考Endaxis中的技能写法，需要实现更精细的技能效果判定系统
- 现有的技能系统只支持基本的技能块显示，缺乏精确的伤害判定点和状态效果显示
- 时序系统对于理解技能的冷却时间和buff持续时间非常重要
- 状态效果系统（如印记）是很多技能的核心机制

## Functional Requirements
- **FR-1**: 实现伤害判定点系统，在技能时间轴上显示精确的伤害触发时刻
- **FR-2**: 实现时序系统，显示buff持续时间条和技能冷却时间条
- **FR-3**: 实现状态效果系统，支持增加或消耗印记等状态效果
- **FR-4**: 在时间轴上可视化显示所有新增元素
- **FR-5**: 保持与现有技能系统的兼容性

## Non-Functional Requirements
- **NFR-1**: 性能要求：时间轴渲染性能不低于现有系统
- **NFR-2**: 可扩展性：系统设计应支持未来添加更多类型的状态效果
- **NFR-3**: 可用性：新增元素的显示应清晰易读，不影响现有功能的使用

## Constraints
- **Technical**: 基于Vue 3 + Vite + Element Plus技术栈
- **Business**: 保持与现有代码风格和架构的一致性
- **Dependencies**: 依赖现有的时间轴组件和技能系统

## Assumptions
- 现有技能数据结构可以扩展，不需要完全重写
- 时间轴组件可以修改以支持新的可视化元素
- 伤害判定点、状态效果和时序系统可以在现有架构基础上实现

## Acceptance Criteria

### AC-1: 伤害判定点系统
- **Given**: 用户在编辑器中选择一个技能
- **When**: 技能包含多个伤害判定点
- **Then**: 时间轴上应显示每个伤害判定点的精确位置
- **Verification**: `human-judgment`
- **Notes**: 判定点应根据技能的frames数据计算得出

### AC-2: 时序系统
- **Given**: 用户在编辑器中使用技能
- **When**: 技能有冷却时间或产生buff
- **Then**: 时间轴上应显示冷却时间条和buff持续时间条
- **Verification**: `human-judgment`
- **Notes**: 冷却时间条应从技能结束开始计算，buff持续时间条应从技能释放开始计算

### AC-3: 状态效果系统
- **Given**: 用户使用带有状态效果的技能
- **When**: 技能增加或消耗印记
- **Then**: 时间轴上应显示状态效果的持续时间和变化
- **Verification**: `human-judgment`
- **Notes**: 状态效果应与技能的marks字段对应

### AC-4: 可视化显示
- **Given**: 用户在时间轴上查看技能
- **When**: 技能包含伤害判定点、buff或状态效果
- **Then**: 所有这些元素应在时间轴上清晰可见
- **Verification**: `human-judgment`
- **Notes**: 不同类型的元素应使用不同的颜色和样式区分

### AC-5: 兼容性
- **Given**: 用户加载使用旧技能数据的项目
- **When**: 打开编辑器查看技能
- **Then**: 旧技能数据应正常显示，不出现错误
- **Verification**: `programmatic`
- **Notes**: 系统应向后兼容旧的技能数据结构
